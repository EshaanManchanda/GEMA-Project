### 3. Form Data Handling ✅

**Location**: `frontend/src/pages/admin/AdminTeachingEditEventPage.tsx`

**Key Transformations:**

#### On Load (from backend):
```tsx
// Backend sends: eventType: 'Online' | 'Offline'
// Frontend uses: teachingMode: 'Online' | 'Offline' | 'Hybrid'
const mappedTeachingMode = eventData.eventType === 'Online' ? 'Online' : 'Offline';
setFormData(prev => ({...prev, teachingMode: mappedTeachingMode}));
```

#### On Submit (to backend):
```tsx
// Frontend has: teachingMode: 'Online' | 'Offline' | 'Hybrid'
// Backend expects: eventType: 'Online' | 'Offline'
// Hybrid is converted to Online
const eventType = formData.teachingMode === 'Hybrid' ? 'Online' : formData.teachingMode;

// Conditional location data
const eventData = {
  eventType: eventType,
  location: {
    country: formData.country || undefined,
    city: formData.city,
    address: (formData.teachingMode === 'Offline') ? formData.address : undefined,
    coordinates: (formData.teachingMode === 'Offline') ? {
      lat: parseFloat(formData.latitude) || 0,
      lng: parseFloat(formData.longitude) || 0
    } : undefined
  },
  meetingLink: (formData.teachingMode === 'Online' || formData.teachingMode === 'Hybrid') 
    ? formData.meetingLink 
    : undefined
};
```

### 4. Validation Rules ✅

Updated validation in `AdminTeachingEditEventPage.tsx`:

```tsx
// Required for all event types
if (!formData.country?.trim()) newErrors.country = 'Country is required';
if (!formData.city?.trim()) newErrors.city = 'City is required';

// Required only for offline
if (formData.teachingMode === 'Offline' && !formData.address?.trim()) {
  newErrors.address = 'Address is required for offline teaching';
}

// Required only for online/hybrid
if ((formData.teachingMode === 'Online' || formData.teachingMode === 'Hybrid') && !formData.meetingLink?.trim()) {
  newErrors.meetingLink = 'Meeting link is required for online/hybrid teaching';
}
```

## 🔄 Data Flow

```
User Creates Teaching Event
        ↓
Basic Info Tab: Select Teacher from Dropdown
        ↓
Choose Teaching Mode (Online/Offline/Hybrid)
        ↓
IF Offline:
  └─ Fill Address, Coordinates, Country, City
IF Online/Hybrid:
  └─ Fill Meeting Link, Country, City (for SEO)
        ↓
Form Submission
        ↓
Transform Data:
  - Convert teachingMode → eventType
  - Include/exclude address based on type
  - Include/exclude meetingLink based on type
  - Keep country/city for all (SEO)
        ↓
Send to Backend:
  eventType: 'Online' | 'Offline'
  location: {country, city, address?, coordinates?}
  meetingLink: string | undefined
  teacherId: string
        ↓
Backend Stores Data
        ↓
Database Updated
```

## 🎓 Key Concepts

### 1. Teacher Assignment
- Every teaching event must have exactly one teacher
- Teachers are selected from a dropdown showing active teachers
- Teacher can be changed anytime
- Backend validates teacher is active and exists

### 2. Event Type Mapping
- Frontend uses `teachingMode` (3 options: Online, Offline, Hybrid)
- Backend uses `eventType` (2 options: Online, Offline)
- Hybrid events are treated as Online on the backend
- This mapping happens transparently in form submission

### 3. Location Data
- **Online/Hybrid**: Country and city are for SEO purposes
- **Offline**: Country, city, and address are for physical location
- Both types require country and city
- Address required only for offline events

### 4. Meeting Link
- Only for Online/Hybrid events
- Can be Zoom, Google Meet, Teams, or any video conferencing URL
- Required for online/hybrid, optional for offline
- Stored in `meetingLink` field
