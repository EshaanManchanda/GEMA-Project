Authentication System

Gema implements a dual authentication system combining JWT tokens for API access with Firebase authentication for social login and identity verification. The system features 4-digit OTP email verification, secure session management, and enterprise-grade security measures.

Dual Auth Methods
4-Digit OTP Codes
10min OTP Expiry
99.9% Security Grade

Dual Authentication Architecture

Why Dual Authentication?
The dual system provides JWT tokens for fast API access and session management, while Firebase handles complex authentication flows like social login, password reset, and cross-platform identity verification.

JWT Token Structure

Secure JSON Web Tokens with role-based claims and expiration management.

JWT Payload Structure
{ "id": "user_id_here", "email": "user@example.com", "role": "customer", "permissions": ["read:events", "write:bookings"], "iat": 1640995200, "exp": 1641599999, "iss": "gema-platform", "aud": "gema-api" }

Token Refresh Strategy

Automatic token refresh with rotation to maintain security without user interruption.

POST /api/auth/refresh-token
Request: Refresh token in body
Response: New access token + new refresh token
Security: Old refresh token invalidated

Implementation Details

OTP Generation

Cryptographically secure 4-digit code generation with collision avoidance.

OTP Generation Logic
// Secure OTP generation const crypto = require('crypto'); function generateOTP() { // Generate cryptographically secure 4-digit code const buffer = crypto.randomBytes(2); const otp = Math.abs(buffer.readInt16BE()) % 10000; return otp.toString().padStart(4, '0'); } // Store with expiration const otpData = { code: generateOTP(), email: userEmail, expiresAt: new Date(Date.now() + 10 * 60 * 1000), // 10 minutes attempts: 0, maxAttempts: 3 };

Secure Storage

OTP codes are hashed and stored with metadata for security and tracking.

Field Type Purpose
hashedCode String Bcrypt hashed OTP
email String Target email address
expiresAt Date 10-minute expiration
attempts Number Failed attempt counter
ipAddress String Security tracking

OTP Verification API

POST /api/auth/verify-email

Verify email using 4-digit OTP code. Includes rate limiting and attempt tracking.

Request Example
POST /api/auth/verify-email Content-Type: application/json Authorization: Bearer { "otp": "1234" } // Success Response { "success": true, "message": "Email verified successfully", "data": { "isEmailVerified": true, "verifiedAt": "2024-01-15T10:30:00Z" } } // Error Response (Invalid OTP) { "success": false, "error": "INVALID_OTP", "message": "Invalid or expired OTP code", "attemptsRemaining": 2 }

Enterprise Security Features

The authentication system implements multiple layers of security to protect user accounts and prevent unauthorized access, meeting enterprise security standards and compliance requirements.

Password Security

Robust password handling with industry-standard encryption and complexity requirements.

Bcrypt Hashing: 12 salt rounds with adaptive cost factor
  • Minimum 8 characters with complexity rules
  • Common password blacklist checking
  • Password history tracking (last 5 passwords)
  • Secure password reset workflows

Rate Limiting

Advanced rate limiting prevents brute force attacks and API abuse.

Endpoint Limit Window
Login 5 attempts 15 minutes
Registration 3 accounts 1 hour
Password Reset 3 requests 1 hour
OTP Resend 5 requests 30 minutes

Session Monitoring

Comprehensive session tracking and anomaly detection for enhanced security.

  • IP Address Tracking: Geographic location monitoring
  • Device Fingerprinting: Browser and device identification
  • Concurrent Sessions: Maximum 3 active sessions per user
  • Suspicious Activity: Automated alerts for unusual patterns

Audit Logging

Complete audit trail for all authentication events and security-related activities.

Logged Events
• User registration and email verification • Login attempts (successful and failed) • Password changes and resets • Account suspensions and reactivations • Token generation and refresh • Permission changes and role updates • Suspicious activity and security alerts

Always implement proper error handling, use HTTPS for all authentication endpoints, store tokens securely on the client side, and implement proper session timeout handling. Regular security audits and penetration testing are essential for maintaining security posture.
