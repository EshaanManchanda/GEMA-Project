REST API Documentation

Comprehensive REST API reference for the Gema Event Management Platform. All endpoints support JSON request/response format with JWT authentication.

API Overview

Base URLs

Development: http://localhost:5000/api
Staging: https://gema-backend-staging.render.com/api
Production: https://gema-backend.render.com/api

Authentication

JWT Bearer tokens required for protected endpoints

Authorization: Bearer <your-jwt-token>

Response Format

All API responses follow consistent JSON format

{
  "success": true,
  "message": "Operation successful",
  "data": { ... }
}

Error Handling

HTTP status codes with detailed error messages

200 - Success
400 - Bad Request
401 - Unauthorized
500 - Server Error

API Endpoints

Authentication

POST /api/auth/register Register a new user account

Request Body

{
  "firstName": "John",
  "lastName": "Doe", 
  "email": "john.doe@example.com",
  "password": "SecurePassword123!",
  "role": "customer",
  "phone": "+1234567890",
  "country": "UAE"
}

Response

{
  "success": true,
  "message": "User registered successfully",
  "data": {
    "user": {
      "id": "user_id",
      "firstName": "John",
      "lastName": "Doe",
      "email": "john.doe@example.com",
      "role": "customer"
    },
    "token": "jwt_token",
    "refreshToken": "refresh_token"
  }
}
POST /api/auth/login Login with email and password

Request Body

{
  "email": "john.doe@example.com",
  "password": "SecurePassword123!"
}
GET /api/auth/me Get current user profile Auth Required
POST /api/auth/logout Logout and invalidate tokens Auth Required

Events

GET /api/events Search and list events with filters

Query Parameters

query
string
Search query for events
category
string
Filter by category ID
city
string
Filter by city
minPrice
number
Minimum price filter
maxPrice
number
Maximum price filter
page
number
Page number (default: 0)
limit
number
Results per page (default: 20)
GET /api/events/:id Get event details by ID
POST /api/vendor/events Create new event (Vendor only) Vendor Auth
PUT /api/vendor/events/:id Update event (Vendor only) Vendor Auth

Error Codes Reference

4xx Client Errors

400
Bad Request - Invalid request format or missing required fields
401
Unauthorized - Authentication required or token invalid
403
Forbidden - Insufficient permissions for this operation
404
Not Found - Requested resource does not exist
422
Unprocessable Entity - Request format is correct but contains invalid data
429
Too Many Requests - Rate limit exceeded

5xx Server Errors

500
Internal Server Error - Unexpected server error occurred
502
Bad Gateway - Invalid response from upstream server
503
Service Unavailable - Server temporarily unavailable

Rate Limiting

General API
1000 requests / hour
Authentication
10 requests / minute
Payment Processing
50 requests / hour
File Upload
20 requests / hour

Rate limits are enforced per IP address and reset every hour. Rate limit information is included in response headers:

X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200
